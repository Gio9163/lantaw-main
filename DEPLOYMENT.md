# LANTAW production deployment

This repository is configured for the following production topology:

- React/Vite frontend on Vercel
- Django REST Framework backend on Render
- Existing Supabase PostgreSQL database
- GitHub repository `Gio9163/lantaw-main`, branch `main`

Never commit real `.env` files, passwords, access tokens, or database connection strings.

## 1. Supabase database

1. Open the existing Supabase project and select **Connect**.
2. Copy the **Session Pooler** connection string. Render services may require the IPv4-compatible pooler instead of Supabase's IPv6-only direct endpoint.
3. Replace the password placeholder locally, but do not write the completed URL into this repository.
4. Keep SSL enabled. Production Django configuration adds `sslmode=require` when it parses `DATABASE_URL`.

The URL has this general form:

```text
postgresql://postgres.PROJECT_REF:PASSWORD@REGION.pooler.supabase.com:5432/postgres
```

If the password contains reserved URL characters, use the URL-encoded password supplied by Supabase's connection panel.

## 2. Render backend

The backend root is `Lantaw-TBD-Backend/lantaw`. The repository-level `render.yaml` configures the service, build, and Gunicorn start command.

1. In Render, create a new Blueprint from `Gio9163/lantaw-main` on `main`.
2. Set these secret/environment values when prompted:

| Variable | Production value |
| --- | --- |
| `DATABASE_URL` | Existing Supabase Session Pooler URL |
| `ALLOWED_HOSTS` | `lantaw-main-bhn6.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | `https://lantaw-main.vercel.app` |
| `CSRF_TRUSTED_ORIGINS` | `https://lantaw-main.vercel.app` |

`SECRET_KEY` is generated by Render and `DEBUG` is fixed to `0` by the blueprint. Multiple hosts or origins must be comma-separated, with no paths or trailing slashes.

The Render build performs:

```bash
python -m pip install -r requirements.txt
python manage.py collectstatic --no-input
python manage.py migrate --no-input
```

The service starts with:

```bash
gunicorn lantaw.wsgi:application --bind 0.0.0.0:$PORT
```

After deployment, confirm that these URLs respond over HTTPS:

- `https://YOUR-RENDER-HOST/api/schema/`
- `https://YOUR-RENDER-HOST/api/projects-public/`

Do not run seed commands against production unless the intended production data has been backed up and the command's effects have been reviewed.

## 3. Vercel frontend

The Vercel project root must be:

```text
Lantaw-TBD-Frontend/lantaw-frontend
```

Use the detected Vite defaults or explicitly configure:

| Setting | Value |
| --- | --- |
| Framework preset | Vite |
| Build command | `npm run build` |
| Output directory | `dist` |
| Production branch | `main` |

Add these Vercel environment variables for Production and Preview as appropriate:

| Variable | Value |
| --- | --- |
| `VITE_API_URL` | `https://lantaw-main-bhn6.onrender.com` |
| `VITE_BASE_PATH` | `/` |

Do not add a trailing slash to `VITE_API_URL`. The frontend client also normalizes one if supplied. Redeploy after changing any `VITE_` variable because Vite embeds these values at build time.

The existing `vercel.json` rewrites client-side routes to `index.html`, so direct navigation and refreshes work with React Router.

## 4. Local development

Backend:

```powershell
cd Lantaw-TBD-Backend/lantaw
Copy-Item .env.example .env
python manage.py migrate
python manage.py runserver
```

Leaving `DATABASE_URL` blank uses local SQLite. Set it only when intentionally connecting local Django to Supabase.

Frontend:

```powershell
cd Lantaw-TBD-Frontend/lantaw-frontend
npm install
npm run dev
```

With no frontend `.env`, Vite proxies relative `/api` calls to `http://127.0.0.1:8000`. To call a different backend directly, copy `.env.example` to `.env.local` and set `VITE_API_URL`.

## 5. Production verification

1. Register and log in through the Vercel domain.
2. Confirm token refresh requests target the Render origin.
3. Verify protected project pages load without CORS or mixed-content errors.
4. Verify direct navigation to a frontend route works after browser refresh.
5. Check Render logs for successful migrations and WhiteNoise static collection.
6. Run `python manage.py check --deploy` with the production environment variables.

If a custom Vercel or Render domain is added later, add its hostname/origin to the corresponding environment variables and redeploy both services.
